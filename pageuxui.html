Pass Path

My specialization in UX/UI Design proposed by CalArts, I created an application prototype (mockup). This application should respond to an user issue.

I choose to respond to a common issue : find the right ticket to buy to do a trip in public transportation. In France, each city system in public transport is different and this is really easy to make a mistake (and get stuck in a station...).


So have an application to guide the user to buy the right ticket for his path seems to be a really good idea ('Find the right ticket' option). To let the user chooses, the application also offer several ticket options (less tickets, cheapest...).

I thinked to another feature : guiding the user to find where to buy the ticket(s) he chooses ('Find a ticket shop' option).

The user can also store the trip he wanted and review them in the 'Stored trips' option.
